Normalization Tutorial

üìö What is Normalization?

Normalization is a database design technique that reduces data redundancy and eliminates undesirable characteristics like Insertion, Update, and Deletion Anomalies.

üîë Key Concepts

  • Functional Dependency (X ‚Üí Y): Value of X uniquely determines value of Y
  • Candidate Key: Minimal set of attributes that can uniquely identify a tuple
  • Prime Attribute: Attribute that is part of any candidate key
  • Superkey: Set of attributes that contains a candidate key

üìä Normal Forms

1NF

All attributes are atomic (no multi-valued attributes)

2NF

1NF + No partial dependencies on candidate key

3NF

2NF + No transitive dependencies

BCNF

Every determinant is a superkey
